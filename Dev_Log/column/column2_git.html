TelpCamp 開発ログ - Git入門編

はじめに：なぜGitを使うのか？

プロジェクト開発を進める中で、「コードを間違って消してしまった」「昨日までは動いていたのに、どこを直したら動かなくなったんだろう？」といった問題に直面することがあります。 Gitは、このような問題を解決するための「バージョン管理システム」です。

Gitを導入することで、プロジェクトの歴史を「セーブポイント（コミット）」として記録し、いつでも過去の状態に戻したり、変更内容を確認したりできるようになります。 これにより、開発者は安心して新しい機能の追加やコードの修正に挑戦できます。このログでは、TelpCampプロジェクトにGitを導入し、その基本的な使い方を学んだ過程をまとめます。

Gitの心臓部：3つのエリア

💡 ワーキングディレクトリ、ステージングエリア、リポジトリ

Gitを理解する上で最も重要なのが、この3つの「場所」の概念です。「荷物の発送」に例えると非常に分かりやすいです。

  • ワーキングディレクトリ: 実際に私たちがコードを編集している作業フォルダ（部屋で品物を準備）。
  • ステージングエリア: コミットしたい変更を一時的に置いておく待機場所（発送する品物を段ボール箱に入れる）。
  • リポジトリ (.git): コミットされた変更履歴が永久に保存されている場所（送り状を貼って発送し、記録が残る）。

この「ステージングエリア」というワンクッションがあるのがGitの大きな特徴です。これにより、関係のない変更がごちゃ混ぜになるのを防ぎ、意味のある単位で変更を記録することができます。

Git基本操作：ターミナル vs VS Code

Gitの基本的な開発サイクルは、コマンドを実行する「ターミナル」と、グラフィカルな「VS Code」の両方で行うことができます。それぞれの操作を比較しながら見ていきましょう。

1. 変更の確認

まず、前回のコミットからどのファイルが変更されたかを確認します。

ターミナル

git statusコマンドを実行します。変更されたファイルが赤字で表示されます。

git status
VS Code

左側の「ソース管理」タブのアイコンに、変更されたファイル数を示すバッジが表示されます。タブを開くと、「変更」セクションにファイルの一覧が表示されます。

2. 差分の確認

具体的にコードのどの部分が変更されたかを確認します。

ターミナル

git diff <ファイル名>コマンドで、指定したファイルの差分を表示します。

git diff index.js
VS Code

「ソース管理」タブでファイル名をクリックすると、左右比較の差分ビューが開きます。また、エディタの行番号の横に表示される色付きの線（青:変更, 緑:追加, 赤:削除）でもリアルタイムに差分を確認できます。

3. 変更をステージング

コミットに含めたい変更を「ステージングエリア」に追加します。

ターミナル

git add <ファイル名>またはgit add .でステージングします。

# 特定のファイルだけ
git add index.js

# すべての変更
git add .
VS Code

「変更」セクションで、ファイル名の右側にある「+」アイコンをクリックします。ファイルが「ステージされている変更」セクションに移動します。

4. 変更をコミット

ステージングした変更を、メッセージを付けてリポジトリに記録します。

ターミナル

git commit -m "メッセージ"コマンドを実行します。

git commit -m "feat: ユーザー登録機能を追加"
VS Code

上部のメッセージボックスにコミットメッセージを入力し、その上にあるチェックマーク(✓)アイコンをクリックします。

5. 履歴の確認

これまでのコミット履歴を確認します。

ターミナル

git logコマンドを使います。--onelineオプションが便利です。

git log --oneline
VS Code

標準機能では少し見づらいですが、「Git Graph」や「GitLens」といった人気の拡張機能を入れると、コミット履歴を非常に見やすいグラフ形式で表示できます。

VS Codeでの差分確認：3つの方法

VS CodeはGitと深く連携しており、様々な方法でコードの差分（変更点）を直感的に確認できます。ここでは主な3つの方法を詳しく解説します。

1. ソース管理ビューでの比較

最も基本的でよく使われる方法です。コミットする前の変更内容全体を確認するのに適しています。

  1. 左側のパネルから「ソース管理」アイコン（枝分かれしたアイコン）をクリックします。
  2. 「変更」または「ステージされている変更」のリストに表示されているファイル名をクリックします。
  3. エディタが左右分割され、左側に変更前（最後のコミット時点）、右側に現在の内容が表示されます。変更箇所はハイライトされ、一目瞭然です。

2. エディタ内のインライン差分表示

コーディングしながらリアルタイムに変更点を確認できる方法です。

コードを編集すると、行番号の横に色付きのインジケーターが表示されます。

  • ■ 青い線: その行が変更されたことを示します。
  • ■ 緑の線: その行が新しく追加されたことを示します。
  • ▼ 赤い三角: その場所から行が削除されたことを示します。

💡 インジケーターの便利な使い方

この色付きの線や三角をクリックすると、ポップアップが表示されます。そこでは、変更前の内容を確認したり、「変更を元に戻す」ボタンでその部分の変更だけを瞬時に取り消したりすることができます。

3. タイムラインビューでの履歴比較

「このファイル、過去のコミットではどうなってたっけ？」と、特定のファイルの歴史を遡りたい時に便利です。

  1. エクスプローラー（ファイル一覧）で差分を見たいファイルを開きます。
  2. エクスプローラーパネルの下部にある「タイムライン」セクションを開きます。
  3. そのファイルのコミット履歴が一覧で表示されます。
  4. リスト内の任意のコミットをクリックすると、そのコミットで行われた変更の差分を現在のコードと比較できます。

これらの機能を使いこなすことで、ターミナルとエディタを行き来することなく、スムーズに開発を進めることができます。

よくあるトラブルシューティング

ターミナルの表示が止まって操作できなくなった時

ターミナルでgit diffやgit logを実行した際、内容がダーッと表示された後にキーボード入力を受け付けなくなることがあります。 これはエラーではなく、Gitがlessという閲覧モード（ページャー）に入っている状態です。

💡 解決策は「q」キー

この閲覧モードを終了するには、キーボードのqキーを押します。 すると、通常のコマンド入力ができる状態に戻ります。

lessモード中は、他にも矢印キーでスクロールしたり、スペースキーでページ送りをしたりできます。 長いログや差分をじっくり確認するための便利な機能ですが、慣れないうちは戸惑うかもしれません。

まとめ：Gitは怖くない、強力な味方

Gitは、どんな変更を加えても「コミット」というセーブポイントさえ作っておけば、いつでも過去の状態に戻ることができる、開発者にとっての強力な安全網です。 最初は覚えることが多く感じるかもしれませんが、基本的なサイクル（add → commit）とVS Codeの連携機能を活用すれば、すぐに日々の開発に欠かせないツールになります。

日々の簡単な操作はVS Codeで直感的に行い、複雑な操作が必要になったらターミナルを使う、というように両者を使い分けることで、開発効率はさらに向上します。 これからも積極的にコミットを作成し、プロジェクトの成長の足跡を記録していきましょう。
