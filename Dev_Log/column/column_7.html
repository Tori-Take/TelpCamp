TelpCamp 開発コラム

コラム7: 面倒なエラー処理をエレガントに - 非同期エラーハンドリング

TelpCampのルートの多くは、データベースとの通信を行うため、async/awaitを使った非同期関数になっています。非同期関数でエラーが発生した場合、それを適切に処理しないとアプリケーションがクラッシュしてしまう可能性があります。

最も基本的な方法は、すべての非同期処理をtry...catchブロックで囲むことですが、これでは同じようなコードが何度も繰り返され、コードが冗長になってしまいます。 

// try...catch を使う場合
router.get('/', async (req, res, next) => {
    try {
        const campgrounds = await Campground.find({});
        res.render('campgrounds/index', { campgrounds });
    } catch (e) {
        next(e); // エラーをExpressのエラーハンドラに渡す
    }
});

この問題を解決するため、TelpCampではcatchAsync（またはwrapAsync）というヘルパー関数を導入しました。

実装の解説

1. ヘルパー関数の作成 (utils/catchAsync.js)

このヘルパー関数は、非同期のルートハンドラ関数を引数として受け取り、その関数を実行して、もしエラー（Promiseがrejectされた場合）が発生したら、それを.catch(next)で捕らえてExpressのエラーハンドリングミドルウェアに渡してくれる、非常に便利な高階関数です。 

// utils/catchAsync.js
module.exports = func => {
    return (req, res, next) => {
        func(req, res, next).catch(next);
    }
}

2. ルートへの適用

このヘルパー関数でルートハンドラをラップするだけで、try...catchブロックを完全に省略できます。コードが非常にクリーンで読みやすくなりました。 

// catchAsync を使う場合
router.get('/', wrapAsync(async (req, res) => {
    const campgrounds = await Campground.find({});
    res.render('campgrounds/index', { campgrounds });
}));

💡 DRY原則とコードの可読性

このcatchAsyncユーティリティは、DRY (Don't Repeat Yourself) 原則の素晴らしい実践例です。繰り返し発生するエラーハンドリングのロジックを一つに集約することで、各ルートは本来のビジネスロジックに集中できるようになり、コード全体の可読性と保守性が劇的に向上しました。
