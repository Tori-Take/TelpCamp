コラム2: Gitによるバージョン管理の導入

プロジェクトの規模が大きくなるにつれて、「いつ、どこを、何のために変更したか」を管理することが重要になります。 そこで、このプロジェクトにGitというバージョン管理システムを導入しました。 このコラムでは、Gitの導入から基本的な使い方までの一連の流れを記録します。

学習の目的

Gitの基本的なワークフローを理解し、日々の開発で自信を持って使えるようになることを目指します。

  • プロジェクトをGitで管理し始める方法を学ぶ (git init)。
  • 変更を記録する基本的なサイクル（add → commit）を習得する。
  • 変更履歴や差分を確認する方法を理解する (git log, git diff)。
  • 「ステージングエリア」というGitの重要な概念を理解する。

ステップ1: Gitリポジトリの作成

まず、TelpCampプロジェクトをGitの管理下に置くための初期設定を行いました。

  1. リポジトリの初期化:

    プロジェクトのルートディレクトリで以下のコマンドを実行し、Gitの管理を開始します。これにより.gitという隠しフォルダが作成され、すべての変更履歴がここに保存されます。 

    git init
  2. .gitignoreファイルの作成:

    Gitに追跡してほしくないファイルやフォルダを指定します。特に、npm installで自動生成されるnode_modulesフォルダはサイズが大きいため、必ず無視するように設定します。 

    # .gitignore の内容
node_modules/
.env
  3. 最初のコミット:

    プロジェクトの初期状態を「最初のコミット」として記録します。

    # すべてのファイルをステージング
git add .

# "Initial commit"というメッセージを付けてコミット
git commit -m "Initial commit"

ステップ2: 日々の開発サイクルと重要な概念

Gitを使った開発は、以下のサイクルの繰り返しです。

  1. コードを編集する。
  2. git statusで変更されたファイルを確認する。
  3. git diffで変更内容の詳細を確認する。
  4. git add .でコミットしたい変更をステージングする。
  5. git commit -m "変更内容のメッセージ"で変更を記録する。
  6. git log --onelineで履歴を確認する。

💡 ワーキングディレクトリ、ステージングエリア、リポジトリ

Gitの操作を理解する上で、この3つの「場所」のイメージが非常に重要です。「荷物の発送」に例えられます。

  • ワーキングディレクトリ: 実際に私たちがコードを編集している作業フォルダ（部屋で品物を準備）。
  • ステージングエリア: コミットしたい変更を一時的に置いておく待機場所（発送する品物を段ボール箱に入れる）。
  • リポジトリ: コミットされた変更履歴が保存されている場所（送り状を貼って発送し、記録が残る）。

git addは、ワーキングディレクトリでの変更を「段ボール箱（ステージングエリア）」に入れる操作です。
git commitは、その段ボール箱に「送り状（コミットメッセージ）」を貼って、「発送（リポジトリに記録）」する操作です。

この2段階のプロセスがあるおかげで、関連する変更だけをまとめて、意味のある単位で履歴を残すことができます。

ステップ3: よく使うコマンドまとめ

git status
現在のプロジェクトの状態（どのファイルが変更されたか、ステージングされているかなど）を確認します。「今どうなってる？」と聞くコマンドです。
git log
これまでのコミット履歴を一覧表示します。--onelineオプションを付けると1行で簡潔に表示されて便利です。
git diff
まだステージングしていない変更の差分を表示します。
git diff --staged
ステージング済みの変更の差分を表示します。コミット直前の最終確認に使います。

ステップ4: VS Codeとの連携

ターミナルでのコマンド操作に加えて、VS Codeに統合されている「ソース管理」機能を使うと、Gitの操作をより直感的に行えることがわかりました。

左側のパネルにあるソース管理タブ（枝分かれしたアイコン）を開くと、GUIで以下の操作が可能です。

  • 変更の確認: 変更されたファイルが一覧表示されます (git statusに相当)。
  • 差分の確認: ファイル名をクリックすると、変更前後の差分が並べて表示されます (git diffに相当)。
  • ステージング: ファイル名の横にある「+」アイコンをクリックすると、そのファイルがステージングされます (git add <ファイル名>に相当)。
  • コミット: 上部のメッセージボックスにコミットメッセージを入力し、チェックマーク(✓)アイコンをクリックするとコミットが完了します (git commitに相当)。

💡 ターミナル vs VS Code

日々の簡単なコミットはVS CodeのGUIで素早く行い、過去のコミットの修正など、少し複雑な操作をしたいときにはターミナルを使う、といった使い分けが非常に効率的です。 実際にこのcolumn2_git.htmlファイル自体のコミットも、VS Codeのソース管理機能を使って行いました。
