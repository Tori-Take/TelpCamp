コラム3: 開発に役立つmongoshコマンド

Webアプリケーションを開発していると、「データは正しくデータベースに保存されているか？」と直接確認したくなる場面がよくあります。 そんな時に強力な味方になるのが、MongoDBをコマンドラインから直接操作できるmongoshです。ここでは、開発中によく使う基本的なコマンドを紹介します。

💡 mongosh 基本操作フロー

通常、以下の流れでデータベースの状態を確認します。

  1. mongoshの起動:

    新しいターミナルを開き、以下のコマンドでMongoDBシェルを起動します。

    mongosh
  2. データベース一覧の表示:

    サーバー上に存在するデータベースの一覧を確認します。

    show dbs
  3. 使用するデータベースの選択:

    TelpCampプロジェクトのデータベースに切り替えます。

    use telp-camp

    switched to db telp-campと表示されれば成功です。

  4. コレクション一覧の表示:

    選択中のデータベースにどのようなコレクション（RDBでいうテーブル）があるか確認します。

    show collections

    campgroundsと表示されるはずです。

  5. データの検索:

    コレクション内のドキュメント（データ）を検索します。

    // コレクション内のドキュメントを全て表示
db.campgrounds.find()

// コレクション内のドキュメントを1件だけ表示
db.campgrounds.findOne()

    findOne()は、データが正しく保存されているか手早く確認したい時に特に便利です。

  6. mongoshの終了:

    確認が終わったら、以下のコマンドでシェルを終了します。

    exit

これらのコマンドを覚えておくと、画面表示がうまくいかない時に、問題がサーバーサイドのロジックにあるのか、それともデータベースのデータ自体にあるのかを切り分けるのに非常に役立ちます。
