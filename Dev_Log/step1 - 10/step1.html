TelpCamp プロジェクト作成の記録

学習の目的

このステップでは、Node.js環境でExpressフレームワークとMongoDBデータベースを使ったWebアプリケーション開発の基礎を学びます。 最終的な目標である「キャンプ場の検索・評価アプリ」の土台を構築することを目指します。

具体的には、以下の項目を達成することを目的とします。

  • Node.jsプロジェクトのセットアップ方法を理解する。
  • Expressを使い、基本的なWebサーバーを起動する方法を学ぶ。
  • Mongooseライブラリを介してMongoDBに接続する方法を習得する。
  • Mongoose Schemaを使い、データベースに保存するデータの構造（モデル）を定義する方法を理解する。
  • EJSテンプレートエンジンを使い、簡単なWebページを表示する方法を学ぶ。

ステップ1: プロジェクトの土台作り

まず、アプリケーションを開発するための基本的な環境を構築し、サーバーを起動できる状態にします。

  1. ターミナル操作による環境構築:

    プロジェクト用のフォルダを作成し、必要なライブラリ（パッケージ）をインストールします。

    # プロジェクト用フォルダを作成して移動
mkdir TelpCamp
cd TelpCamp

# Node.jsプロジェクトを初期化
npm init -y

# 必要なパッケージをインストール
npm install express mongoose ejs method-override

# コードを整理するためのフォルダを作成
mkdir models views

    💡 インストールしたパッケージの役割

    • express: Webサーバーを簡単に構築するためのフレームワーク。
    • mongoose: MongoDBをJavaScriptのオブジェクトのように直感的に操作するためのライブラリ。
    • ejs: HTMLの中にJavaScriptの変数を埋め込み、動的なページを作るためのテンプレートエンジン。
    • method-override: HTMLフォームでPUTやDELETEリクエストを使えるようにするためのライブラリ（後のCRUD実装で使用）。
  2. モデルの作成 (models/campground.js):

    データベースに保存する「キャンプ場」のデータ構造（スキーマ）を定義します。名前、場所、料金は必須項目とし、紹介コメントは任意項目としました。

    const mongoose = require('mongoose');
const Schema = mongoose.Schema;

// キャンプ場のデータ構造を定義します
const CampgroundSchema = new Schema({
    name: {
        type: String,
        required: true // 必須項目
    },
    location: {
        type: String,
        required: true // 必須項目
    },
    price: {
        type: Number,
        required: true // 必須項目
    },
    description: {
        type: String
        // required: true を付けないので、この項目は任意になります
    }
});

// 定義したスキーマを元にモデルを作成し、エクスポートします
module.exports = mongoose.model('Campground', CampgroundSchema);

    💡 Mongooseスキーマとは？

    データベースに保存するデータの「設計図」のようなものです。各データ（フィールド）がどのような型（文字列、数値など）を持つか、必須項目（required: true）かどうかなどを定義できます。これにより、意図しないデータが保存されるのを防ぎ、アプリケーションの安定性を高めます。

  3. Expressサーバーの作成 (index.js):

    Webサーバーを起動し、MongoDBに接続し、リクエストに応じてトップページを表示するメインのプログラムを作成します。

    // 必要なパッケージを読み込みます
const express = require('express');
const app = express();
const path = require('path');
const mongoose = require('mongoose');
const methodOverride = require('method-override');

// 作成したモデルを読み込みます
const Campground = require('./models/campground');

// --- データベース接続 ---
async function connectDB() {
  try {
    await mongoose.connect('mongodb://127.0.0.1:27017/telp-camp');
    console.log("MongoDBに接続しました。");
  } catch (err) {
    console.error("MongoDB接続エラー:", err);
  }
}
connectDB();

// --- Expressの設定 ---
app.set('view engine', 'ejs');
app.set('views', path.join(__dirname, 'views'));
app.use(express.urlencoded({ extended: true }));
app.use(methodOverride('_method'));

// --- ルーティング ---
app.get('/', (req, res) => {
    res.render('home');
});

// --- サーバー起動 ---
const port = 3000;
app.listen(port, () => {
  console.log(`ポート${port}で待機中...`);
});

    💡 データベースへの接続と async/await

    データベースへの接続は、完了までに時間がかかる「非同期処理」です。そのため、async/await構文を使って処理が終わるのを待つのが一般的です。また、接続は失敗する可能性があるので、try...catch構文でエラーを捕捉し、プログラムが停止するのを防ぎます。この処理をサーバー起動時に行うことで、アプリケーションがリクエストを受け付ける前にデータベースとの接続を確立できます。

  4. サーバーの起動と確認:

    ターミナルで以下のコマンドを実行し、サーバーを起動します。

    node index.js

    ブラウザで http://localhost:3000 にアクセスし、「TelpCampへようこそ！」というトップページが表示されることを確認します。
