TelpCamp プロジェクト作成の記録

学習の目的

詳細ページに続き、キャンプ場の一覧ページのデザインも改善します。また、ダミーデータの画像が毎回変わってしまう問題を解決し、各キャンプ場に固定の画像が割り当てられるようにします。

  • Bootstrapのグリッドシステム(row, col-*)とCardコンポーネントを組み合わせ、複雑なレイアウトを構築する方法を学ぶ。
  • 外部画像サービスの仕様を理解し、意図した通りのデータを取得するためのURLの指定方法を学ぶ。
  • アプリケーションのデータの一貫性（同じデータは常に同じように表示されるべき）の重要性を理解する。

ステップ12: 一覧ページのデザイン改善と画像URLの永続化

詳細ページとデザインを統一し、より見やすい一覧ページを作成しました。また、開発体験を向上させるための重要な修正も行いました。

  1. 一覧ページのデザイン改善 (views/campgrounds/index.ejs):

    これまでのシンプルなリスト表示から、BootstrapのグリッドシステムとCardを組み合わせたレイアウトに変更しました。左に画像、右にテキスト情報が配置される横長のカード形式です。

    <% for (let campground of campgrounds) { %>
    <div class="card mb-3">
        <div class="row g-0">
            <div class="col-md-4">
                <img src="<%= campground.image %>" class="img-fluid rounded-start" alt="...">
            </div>
            <div class="col-md-8">
                <div class="card-body">
                    <h5 class="card-title"><%= campground.name %></h5>
                    ...
                    <a href="/campgrounds/<%= campground._id %>" class="btn btn-primary">詳細を見る</a>
                </div>
            </div>
        </div>
    </div>
<% } %>

    💡 グリッドシステムによるレイアウト

    • row: グリッドレイアウトの「行」を定義します。
    • col-md-4 / col-md-8: 行の中の「列」を定義します。mdは中くらいの画面サイズ(medium)を表し、画面幅が一定以上の場合に、12分割されたうちの4つ分と8つ分の幅をそれぞれ取るように指定しています。これにより、画像とテキストが横に並びます。
    • img-fluid: 画像が親要素（この場合はcol-md-4）の幅に合わせて自動でリサイズされるようになり、レイアウト崩れを防ぎます。
  2. 画像URLの永続化 (seeds/seedindex.js):

    一覧ページと詳細ページで画像が異なったり、リロードするたびに画像が変わってしまう問題に気づきました。これは、ダミー画像サービスpicsum.photosの仕様が原因でした。

    💡 問題の原因と解決策

    原因: 当初使用していたhttps://picsum.photos/800/600?random=${i}というURLは、「アクセスするたびにランダムな画像を取得する」という仕様でした。そのため、ページが表示されるたびに毎回違う画像が読み込まれていました。

    解決策: URLの形式をhttps://picsum.photos/id/${i}/800/600に変更しました。この形式は、指定したIDに紐づく固定の画像を返す仕様です。ループのインデックスiをIDとして使うことで、各キャンプ場にユニークで、かつ何度アクセスしても変わらない画像を割り当てることができました。

    この修正により、アプリケーションのデータの一貫性が保たれ、より自然な動作になりました。

    // seeds/seedindex.js の修正
const camp = new Campground({
    // ...
    image: `https://picsum.photos/id/${i}/800/600`,
    // ...
});
  3. 動作確認:

    node seeds/seedindex.jsコマンドでデータベースのデータを再投入し、サーバーを再起動しました。 ブラウザで確認したところ、一覧ページと詳細ページの画像がキャンプ場ごとに一致し、ページをリロードしても画像が変わらないことが確認できました。
