TelpCamp プロジェクト作成の記録

学習の目的

CRUD機能の「C (Create)」に対応する、ユーザーが新しいデータを入力するためのフォームページを作成し、デザインを整えます。

  • Bootstrapのフォーム関連クラスを使い、見やすく使いやすい入力フォームを作成する方法を学ぶ。
  • Bootstrapのグリッドシステムを使い、フォームをページの中央に配置するなど、レイアウトを制御する方法を習得する。
  • フォームのname属性の命名規則と、それがExpressサーバー側でどのようにデータとして受け取られるか（req.bodyの構造）を理解する。

ステップ13: 新規登録ページの作成とデザイン

キャンプ場を新しく登録するためのフォームページ（new.ejs）を作成し、Bootstrapを使って見た目を整えました。

  1. 新規登録ページの作成 (views/campgrounds/new.ejs):

    views/campgroundsフォルダにnew.ejsファイルを新規作成しました。このフォームは、POSTメソッドで/campgroundsにデータを送信します。 

    <% layout('layouts/boilerplate') %>

<div class="row">
    <h1 class="text-center">新しいキャンプ場を登録</h1>
    <div class="col-6 offset-3">
        <form action="/campgrounds" method="POST">
            <div class="mb-3">
                <label class="form-label" for="name">キャンプ場名</label>
                <input class="form-control" type="text" id="name" name="campground[name]">
            </div>
            <!-- ... 他の入力項目 ... -->
            <div class="mb-3">
                <label class="form-label" for="price">価格</label>
                <div class="input-group">
                    <span class="input-group-text">¥</span>
                    <input type="number" id="price" class="form-control" name="campground[price]">
                    <span class="input-group-text">/泊</span>
                </div>
            </div>
            <div class="mb-3">
                <button class="btn btn-success">登録する</button>
            </div>
        </form>
        <a href="/campgrounds">一覧へ戻る</a>
    </div>
</div>
  2. デザインとレイアウトの解説:

    Bootstrapのクラスを活用して、フォームのデザインと配置を整えました。

    • 中央配置: <div class="col-6 offset-3">を使い、フォームを画面の中央に配置しました。これは「画面を12分割したうちの6つ分の幅を使い、左側に3つ分の空白を設ける」という意味です。
    • フォーム部品: form-labelやform-controlといったクラスで、ラベルと入力欄のスタイルを統一しました。
    • インプットグループ: 価格の入力欄ではinput-groupを使い、「¥」マークや「/泊」といったテキストを入力欄と一体化させて表示しました。

  3. 💡 name="campground[...]"の重要性

    各<input>タグのname属性をcampground[name]やcampground[location]のように設定しました。 これは非常に重要なテクニックです。

    この命名規則により、フォームが送信されたとき、Expressサーバーはreq.bodyを以下のようなネストされたオブジェクトとして自動的に解析してくれます。 

    // req.body の中身の例
{
  campground: {
    name: '入力されたキャンプ場名',
    location: '入力された場所',
    price: '入力された価格'
    // ...
  }
}

    このおかげで、サーバー側のPOSTルートではnew Campground(req.body.campground)と書くだけで、Mongooseモデルに必要なデータをまとめて渡すことができ、コードが非常に簡潔になります。
