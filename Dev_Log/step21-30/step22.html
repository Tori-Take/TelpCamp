Step 22: レビューの削除機能実装

レビューの投稿・表示機能に続き、投稿されたレビューを削除する機能を実装しました。 これにより、ユーザーは不要になったレビューを削除でき、アプリケーションのデータ管理能力が向上します。

学習の目的

  • 関連付けられた2つのモデル（CampgroundとReview）のデータを整合性を保ちながら削除する方法を学ぶ。
  • MongoDBの$pull演算子を使い、配列フィールドから特定の要素を削除する方法を理解する。
  • RESTfulなルーティングに基づき、ネストされたリソース（キャンプ場に属するレビュー）の削除エンドポイントを設計・実装する。

実装の解説

実装は、フロントエンド（ビュー）に削除ボタンを追加し、バックエンド（ルート）で削除処理を実装する2段階で行います。

  1. 1. 削除ボタンの追加 (views/campgrounds/show.ejs)

    まず、キャンプ場詳細ページで各レビューを表示している部分に、削除用のフォームを追加します。 どのキャンプ場のどのレビューを削除するのかをサーバーに伝えるため、URLに両方のIDを含めることが重要です。 

    <div class="card-body">
    <h5 class="card-title">評価: <%= review.rating %></h5>
    <p class="card-text"><%= review.body %></p>
    <!-- ↓ このフォームを追加 ↓ -->
    <form action="/campgrounds/<%= campground._id %>/reviews/<%= review._id %>?_method=DELETE" method="POST">
        <button class="btn btn-sm btn-danger">削除</button>
    </form>
</div>

    キャンプ場削除の時と同様に、method-overrideライブラリを利用して、このPOSTリクエストをDELETEリクエストとして扱います。

  2. 2. 削除ルートの実装 (index.js)

    次に、上記のフォームから送信されたリクエストを処理するルートをindex.jsに追加します。 この処理は、単にレビューを削除するだけでは不十分です。 

    // レビュー削除処理のDELETEリクエスト
app.delete('/campgrounds/:id/reviews/:reviewId', wrapAsync(async (req, res) => {
    const { id, reviewId } = req.params;
    // 1. Campgroundモデルのreviews配列から、該当するreviewIdを削除する
    await Campground.findByIdAndUpdate(id, { $pull: { reviews: reviewId } });
    // 2. Reviewモデルから、該当するレビュー本体を削除する
    await Review.findByIdAndDelete(reviewId);
    res.redirect(`/campgrounds/${id}`);
}));

    💡 $pull演算子による関連データの削除

    今回の実装の核心部分です。レビューを削除するには、2つの操作が必要です。

    1. レビュー本体の削除: Review.findByIdAndDelete(reviewId)で、reviewsコレクションからレビューのドキュメントそのものを削除します。
    2. 関連情報の削除: レビュー本体を消しても、Campgroundドキュメントのreviews配列には、削除されたレビューのIDが「ゴミ」として残ってしまいます。これを放置すると、存在しないデータを参照しようとしてエラーの原因になります。
      そこで、MongoDBの$pull演算子を使います。Campground.findByIdAndUpdate(id, { $pull: { reviews: reviewId } })は、「IDがidのキャンプ場のreviews配列から、reviewIdと一致する値を取り除いて更新する」という操作を行います。

    この2つの操作を両方行うことで、データベースの整合性を保ったまま、レビューを完全に削除することができます。

これで、キャンプ場とレビューの基本的なCRUD（作成、読み取り、更新、削除）機能が一通り完成しました。
