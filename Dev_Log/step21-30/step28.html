Step 28: フラッシュメッセージの拡充とエラーハンドリング改善

前回実装したフラッシュメッセージ機能を、キャンプ場のすべてのCRUD操作（作成、更新、削除）とレビューの操作に適用しました。 さらに、存在しないキャンプ場にアクセスしようとした際のエラーハンドリングを改善し、よりユーザーフレンドリーなフィードバックを提供するように変更しました。

学習の目的

  • 既存のフラッシュメッセージ機能を他のルート（更新、削除）にも展開し、コードの再利用性を確認する。
  • エラーハンドリングの一環としてフラッシュメッセージを活用する方法を学ぶ。
  • ユーザー体験（UX）を考慮し、技術的なエラーページではなく、状況を説明するメッセージと適切なリダイレクトでユーザーを導く方法を実践する。

実装の解説

主にルーターファイル（routes/campgrounds.js, routes/reviews.js）を修正しました。

  1. 1. 更新・削除ルートへのフラッシュメッセージ追加

    キャンプ場の更新（Update）および削除（Delete）ルートで、処理が成功した後にリダイレクトする直前に、成功メッセージをセットしました。 

    // routes/campgrounds.js

// Update
router.put('/:id', ..., wrapAsync(async (req, res) => {
    // ... 更新処理 ...
    req.flash('success', 'キャンプ場情報を更新しました。');
    res.redirect(`/campgrounds/${campground._id}`);
}));

// Delete
router.delete('/:id', wrapAsync(async (req, res) => {
    // ... 削除処理 ...
    req.flash('success', 'キャンプ場情報を削除しました。');
    res.redirect('/campgrounds');
}));

    これにより、ユーザーは自分の操作が正常に完了したことを明確に認識できます。

  2. 2. 「Not Found」エラーハンドリングの改善

    存在しないIDのキャンプ場詳細ページや編集ページにアクセスしようとした際の挙動を変更しました。 これまでは汎用的なエラーページを表示していましたが、代わりにエラーメッセージをフラッシュで表示し、キャンプ場一覧ページへリダイレクトするようにしました。 

    // routes/campgrounds.js の Show ルート
router.get('/:id', wrapAsync(async (req, res, next) => {
    const campground = await Campground.findById(req.params.id).populate('reviews');
    if (!campground) {
        // 変更前: return next(new ExpressError('...', 404));
        // 変更後:
        req.flash('error', '指定されたIDのキャンプ場は見つかりませんでした。');
        return res.redirect('/campgrounds');
    }
    res.render('campgrounds/show', { campground });
}));

    💡 ユーザーフレンドリーなエラーハンドリング

    この変更は、ユーザー体験を大きく向上させます。ユーザーは「ページが見つかりません」という技術的なエラーに直面するのではなく、「探しているキャンプ場は見つかりませんでした」という具体的な状況を理解し、自然に一覧ページに戻ることができます。 このように、エラーの種類に応じて適切なフィードバックと導線を提供することは、質の高いアプリケーションを開発する上で非常に重要です。

  3. 3. レビュー関連のフラッシュメッセージ追加・修正

    同様に、レビューの作成・削除時にもフラッシュメッセージを追加・修正し、操作結果がフィードバックされるようにしました。 

    // routes/reviews.js

// Create Review
router.post('/', ..., wrapAsync(async (req, res) => {
    // ... レビュー作成処理 ...
    req.flash('success', '新しいレビューを作成しました。');
    res.redirect(`/campgrounds/${campground._id}`);
}));

// Delete Review
router.delete('/:reviewId', wrapAsync(async (req, res) => {
    // ... レビュー削除処理 ...
    req.flash('success', 'レビューを削除しました。');
    res.redirect(`/campgrounds/${id}`);
}));

    これで、アプリケーション内の主要な操作のほとんどで、ユーザーに分かりやすいフィードバックが返されるようになりました。
