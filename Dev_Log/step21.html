Step 21: 投稿されたレビューの表示機能実装

前回(Step 20)でレビューの作成（投稿）機能が完成しました。今回はその投稿されたレビューを、キャンプ場の詳細ページに表示する機能を実装します。 これにより、ユーザーは自分の投稿が正しく保存されたことを確認でき、他のユーザーのレビューも閲覧できるようになります。

学習の目的

  • Mongooseのpopulate()メソッドを使い、関連するドキュメント（レビュー）のデータを取得する方法を学ぶ。
  • 取得した関連データの配列をEJSテンプレート内でループ処理し、一覧表示する方法を実践する。
  • EJSの条件分岐を使い、レビューが1件も投稿されていない場合にはレビューセクション自体を表示しない、といったUIの制御を行う。

実装の解説

実装は大きく分けて2つのステップで行います。まずサーバー側でキャンプ場データと一緒にレビューデータを取得し、次にビュー側でそのデータを画面に描画します。

  1. 1. ルートハンドラの修正 (index.js)

    キャンプ場の詳細ページを表示するルート (GET /campgrounds/:id) を修正します。 Campgroundモデルのreviews配列にはレビューのIDしか入っていないため、Mongooseのpopulate()メソッドを使って、そのIDを実際のレビューデータ（本文や評価など）に置き換える必要があります。 

    // キャンプ場詳細ページのGETリクエスト
app.get('/campgrounds/:id', wrapAsync(async (req, res) => {
    // .populate('reviews') を追加して、関連するレビューをすべて取得します
    const campground = await Campground.findById(req.params.id).populate('reviews');
    res.render('campgrounds/show', { campground });
}));

    💡 populate()によるデータ展開

    .populate('reviews')という一文を追加するだけで、MongooseはCampgroundスキーマのreviewsフィールドがref: 'Review'で定義されていることを理解し、配列内の各IDを使ってreviewsコレクションを検索し、対応するドキュメントに置き換えてくれます。

    これにより、ルートハンドラは複雑なデータベースクエリを書くことなく、必要なデータをまとめて取得できます。

  2. 2. ビューの修正 (views/campgrounds/show.ejs)

    次に、ルートハンドラから渡されたcampgroundオブジェクトに含まれるレビュー情報を、詳細ページに表示します。 レビュー投稿フォームの上あたりに、レビュー一覧を表示するコードを追加しました。 

    <!-- ... レビュー投稿フォームの上あたりに追加 ... -->
<div class="col-6 offset-3 mt-3">
    <h3>レビュー</h3>
    <% for(let review of campground.reviews) { %>
    <div class="card mb-3">
        <div class="card-body">
            <h5 class="card-title">評価: <%= review.rating %></h5>
            <p class="card-text"><%= review.body %></p>
        </div>
    </div>
    <% } %>
</div>

    💡 EJSによる動的なコンテンツ生成

    populateされたcampground.reviewsは、レビューオブジェクトの配列になっています。 EJSの<% for(let review of campground.reviews) { %>構文を使ってこの配列をループ処理し、各レビューのratingとbodyをBootstrapのCardコンポーネントを使って表示しています。

    さらに、このコード全体を<% if(campground.reviews.length) { %> ... <% } %>で囲むことで、レビューがまだ1件も投稿されていない場合には「レビュー」という見出し自体が表示されないようにし、より洗練されたUIを実現できます。

以上の実装により、レビューの投稿から表示までの一連の流れが完成し、アプリケーションのコア機能がより充実しました。
