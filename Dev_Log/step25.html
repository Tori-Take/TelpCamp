Step 25: 静的ファイルの配信とスクリプトの外部化

これまでレイアウトファイル（boilerplate.ejs）に直接記述していたBootstrapのフォームバリデーション用JavaScriptを、外部ファイルに分離しました。 そして、Expressの静的ファイル配信機能（express.static）を使って、その外部ファイルをクライアントに提供するように設定しました。

学習の目的

  • express.staticミドルウェアを使い、特定のディレクトリ（例: public）を静的ファイル配信用に設定する方法を学ぶ。
  • HTMLファイルからJavaScriptを分離することで、コードの関心事を分離し、可読性と保守性を向上させる。
  • 外部スクリプトの利点（ブラウザキャッシュによるパフォーマンス向上、コードの再利用性）を理解する。

実装の解説

実装は3つのステップで行いました。

  1. 1. 静的ファイル用のディレクトリとJSファイルの作成

    まず、プロジェクトのルートにpublicフォルダを作成し、その中にjavascriptsフォルダ、さらにその中にvalidation.jsファイルを作成しました。 そして、boilerplate.ejsにあったスクリプトをこの新しいファイルに移動させました。 

    // public/javascripts/validation.js
(() => {
    'use strict'
    const forms = document.querySelectorAll('.needs-validation')
    Array.from(forms).forEach(form => {
        form.addEventListener('submit', event => {
            if (!form.checkValidity()) {
                event.preventDefault()
                event.stopPropagation()
            }
            form.classList.add('was-validated')
        }, false)
    })
})()

  2. 2. Expressに静的ファイルディレクトリを教える (index.js)

    次に、index.jsにexpress.staticミドルウェアを追加し、Expressにpublicフォルダを静的ファイル置き場として認識させました。 

    // index.js
const path = require('path');

// ...

// publicディレクトリの静的ファイルを提供するための設定
app.use(express.static(path.join(__dirname, 'public')));

// ...

    💡 express.staticの仕組み

    この一行を追加することで、ExpressはHTTPリクエストを受け取った際に、まずpublicフォルダ内に一致するファイルがないかを探すようになります。 例えば、ブラウザが/javascripts/validation.jsをリクエストすると、Expressはpublic/javascripts/validation.jsという物理ファイルを探し、見つかればその内容を直接クライアントに返します。URLにpublicは含まれないのがポイントです。

  3. 3. レイアウトファイルの修正 (boilerplate.ejs)

    最後に、boilerplate.ejsから元の<script>ブロックを削除し、代わりに外部ファイルを読み込むための1行に置き換えました。 

    <!-- ... -->
<body>
    <!-- ... -->
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js" ...></script>

    <!-- カスタムバリデーションスクリプトの読み込み -->
    <script src="/javascripts/validation.js"></script>
</body>
<!-- ... -->

    この変更により、boilerplate.ejsは純粋なHTML構造に近づき、JavaScriptのロジックはvalidation.jsに集約され、コード全体がよりクリーンで管理しやすくなりました。
