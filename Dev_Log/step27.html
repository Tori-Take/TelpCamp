TelpCamp プロジェクト作成の記録

Step 27: フラッシュメッセージの実装

ユーザーがキャンプ場の作成などの操作を行った後、リダイレクトを挟んで「作成しました」といったフィードバックメッセージを表示する「フラッシュメッセージ」機能を実装しました。 この機能はセッションを利用しており、一度表示されると消える一時的なメッセージです。

学習の目的

  • connect-flashパッケージを使い、フラッシュメッセージを管理する方法を学ぶ。
  • req.flash('key', 'message')でメッセージを設定し、req.flash('key')で取り出す仕組みを理解する。
  • res.localsとカスタムミドルウェアを組み合わせ、すべてのテンプレートで共通の変数を使えるようにする方法を学ぶ。
  • EJSのパーシャル（部分テンプレート）機能を使い、UIコンポーネントを再利用可能な形で管理する方法を実践する。

実装の解説

実装は、ミドルウェアの設定、ルートでのメッセージ設定、ビューでの表示という3段階で行いました。

  1. 1. ミドルウェアの設定 (index.js)

    まずnpm install connect-flashでパッケージをインストールし、index.jsでミドルウェアを有効化しました。 特に重要なのは、すべてのリクエストでフラッシュメッセージを変数としてビューに渡すためのカスタムミドルウェアです。 

    // index.js
const flash = require('connect-flash');
// ...
app.use(session(sessionConfig));
app.use(flash()); // sessionミドルウェアの後に設定

// すべてのリクエストでフラッシュメッセージをビューで使えるようにするミドルウェア
app.use((req, res, next) => {
    res.locals.success = req.flash('success');
    res.locals.error = req.flash('error');
    next();
});

    💡 res.localsによるグローバル変数

    res.localsは、そのリクエスト〜レスポンスのサイクル中だけ存在するオブジェクトです。ここに格納されたプロパティ（例: res.locals.success）は、レンダリングされるEJSテンプレート内でグローバル変数（例: success）のように直接アクセスできます。 このミドルウェアのおかげで、どのテンプレートからでもsuccessやerrorといった変数を参照できるようになります。

  2. 2. ルートでのメッセージ設定 (routes/campgrounds.js)

    キャンプ場が正常に作成された後、リダイレクトする直前にreq.flash()を使ってメッセージをセッションに保存します。 

    // routes/campgrounds.js のCreateルート
router.post('/', validateCampground, wrapAsync(async (req, res) => {
    const campground = new Campground(req.body.campground);
    await campground.save();
    req.flash('success', '新しいキャンプ場を作成しました。'); // ここでメッセージを設定
    res.redirect(`/campgrounds/${campground._id}`);
}));

  3. 3. ビューでのメッセージ表示

    メッセージ表示部分を再利用可能なパーシャルファイルviews/partials/flash.ejsとして作成しました。 

    <!-- views/partials/flash.ejs -->
<% if(success && success.length) { %>
    <div class="alert alert-success alert-dismissible fade show" role="alert">
        <%= success %>
        <button type="button" class="btn-close" data-bs-dismiss="alert" aria-label="Close"></button>
    </div>
<% } %>

<% if(error && error.length) { %>
    <div class="alert alert-danger alert-dismissible fade show" role="alert">
        <%= error %>
        <button type="button" class="btn-close" data-bs-dismiss="alert" aria-label="Close"></button>
    </div>
<% } %>

    そして、このパーシャルを共通レイアウトファイルviews/layouts/boilerplate.ejsに組み込むことで、すべてのページでフラッシュメッセージが表示される準備が整いました。 

    <!-- views/layouts/boilerplate.ejs -->
<main class="container mt-5">
    <%- include('../partials/flash') %>
    <%- body %>
</main>

    この構成により、メッセージが存在する場合のみ、Bootstrapのスタイルが適用されたアラートが表示されるようになりました。
