TelpCamp プロジェクト作成の記録

Step 30: ユーザー登録機能の実装

Passportの認証基盤を元に、実際のユーザー登録画面と、フォームから送信されたデータを処理するロジックを実装しました。

学習の目的

  • ユーザー登録用のフォームと、それに関連するルートを専用のルーターファイルに分離する。
  • passport-local-mongooseが提供するUser.register(user, password)メソッドを使い、ユーザーの登録処理を実装する。
  • 登録処理中にエラー（ユーザー名の重複など）が発生した場合のハンドリング方法を学ぶ (try...catch)。
  • 登録成功後、req.login()メソッドを使ってユーザーを自動的にログイン状態にする方法を習得する。
  • connect-flashを使い、登録の成功・失敗をユーザーにフィードバックする。

実装の解説

ユーザー関連のルートをまとめるため、新しくroutes/users.jsを作成し、登録フォームの表示と登録処理のロジックを実装しました。

1. ユーザー登録フォームの作成 (views/users/register.ejs)

ユーザー名、メールアドレス、パスワードを入力するためのフォームを作成しました。Bootstrapのクラスを使ってスタイリングし、クライアントサイドのバリデーションも設定しています。 

<% layout('layouts/boilerplate') %>

<h1>ユーザー登録</h1>
<form action="/register" method="POST" class="validated-form" novalidate>
    <div class="mb-3">
        <label class="form-label" for="username">ユーザー名</label>
        <input class="form-control" type="text" id="username" name="username" required>
    </div>
    <!-- email, passwordの入力欄 ... -->
    <button class="btn btn-success">登録</button>
</form>

2. ユーザー登録ルートの作成 (routes/users.js)

登録フォームを表示するGETルートと、フォームデータを受け取って処理するPOSTルートを定義しました。 

router.post('/register', catchAsync(async (req, res, next) => {
    try {
        const { email, username, password } = req.body;
        const user = new User({ email, username });
        // User.registerでユーザーを登録
        const registeredUser = await User.register(user, password);
        // req.loginで自動ログイン
        req.login(registeredUser, err => {
            if (err) return next(err);
            req.flash('success', 'TelpCampへようこそ！');
            res.redirect('/campgrounds');
        });
    } catch (e) {
        req.flash('error', e.message);
        res.redirect('/register');
    }
}));

💡 User.register()とreq.login()

  • User.register(user, password): passport-local-mongooseが提供する中心的なメソッドです。ユーザーオブジェクトと平文のパスワードを渡すと、パスワードのハッシュ化とソルトの生成、そしてデータベースへの保存までをすべて行ってくれます。ユーザー名が既に存在する場合はエラーをスローします。
  • req.login(user, callback): Passportがreqオブジェクトに追加するメソッドです。ユーザー登録が成功した直後にこのメソッドを呼び出すことで、ユーザーをログイン状態にし、セッションを確立できます。これにより、ユーザーは登録後に再度ログインする手間が省けます。

3. メインファイルへのルーターの登録 (index.js)

作成したユーザールーターを、アプリケーションのメインファイルであるindex.jsに登録しました。 

const userRoutes = require('./routes/users');
// ...
app.use('/', userRoutes);

これで、ユーザーは/registerにアクセスして新しいアカウントを作成できるようになりました。

4. (改善) エラーメッセージの日本語化

ユーザー登録時に発生するエラーメッセージ（ユーザー名の重複など）は、デフォルトでは英語で表示されます。これをユーザーにとって分かりやすい日本語のメッセージに変更し、UX（ユーザー体験）を向上させました。

models/user.jsファイルでpassport-local-mongooseプラグインを適用する際に、errorMessagesオプションを渡すことでメッセージを上書きします。 

// models/user.js
UserSchema.plugin(passportLocalMongoose, {
    errorMessages: {
        UserExistsError: 'そのユーザー名はすでに使用されています。',
        MissingPasswordError: 'パスワードが入力されていません。',
        IncorrectPasswordError: 'パスワードまたはユーザー名が正しくありません。',
        IncorrectUsernameError: 'パスワードまたはユーザー名が正しくありません。',
        // ...その他のエラーメッセージ
    }
});

💡 セキュリティを考慮したメッセージ

IncorrectPasswordError（パスワード間違い）とIncorrectUsernameError（ユーザー名間違い）のメッセージを、意図的に同じ「パスワードまたはユーザー名が正しくありません。」に統一しています。これは、「そのユーザー名は存在するが、パスワードが違う」といった具体的な情報を攻撃者に与えないための、一般的なセキュリティプラクティスです。

この修正により、ルートのcatchブロックで捕捉されるエラーのe.messageが、ここで設定した日本語のメッセージに置き換わります。
