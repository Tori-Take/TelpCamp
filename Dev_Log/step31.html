Step 31: ログイン・ログアウト機能の実装

ユーザー登録機能に続き、認証システムの核となるログイン、ログアウト機能を実装しました。Passportの強力なミドルウェアを活用することで、少ないコードで安全な認証処理を実現しています。

学習の目的

  • passport.authenticate()ミドルウェアを使い、ログイン処理を実装する方法を学ぶ。
  • 認証の成功時と失敗時（リダイレクト、フラッシュメッセージ）の処理をPassportに任せる方法を理解する。
  • req.logout()メソッドを使い、セッションを破棄してログアウトさせる方法を習得する。
  • req.userとres.localsを使い、ログイン状態をビュー（EJS）に渡し、UIを動的に変更する方法を学ぶ。
  • ログイン中のユーザー名を表示し、パーソナライズされたUIを提供する。

実装の解説

ログインフォームの作成、ルートの追加、そしてUIの更新という流れで実装しました。

1. ログイン・ログアウトのルート追加 (routes/users.js)

ログインフォームを表示するGETルート、ログイン処理を行うPOSTルート、そしてログアウト用のGETルートをusers.jsに追加しました。 

// ログイン処理
router.post('/login', passport.authenticate('local', { failureFlash: true, failureRedirect: '/login' }), (req, res) => {
    req.flash('success', 'おかえりなさい！');
    res.redirect('/campgrounds');
});

// ログアウト処理
router.get('/logout', (req, res, next) => {
    req.logout(function (err) {
        if (err) { return next(err); }
        req.flash('success', 'ログアウトしました。');
        res.redirect('/campgrounds');
    });
});

💡 passport.authenticate()ミドルウェア

この一行がPassportの強力さを示しています。これはミドルウェアとして機能し、認証に関する定型的な処理をすべて引き受けてくれます。

  • フォームから送信されたusernameとpasswordを自動で取得します。
  • passport-local-mongooseが提供する認証ロジックを呼び出します。
  • 認証に失敗すると、failureRedirectで指定されたURLにリダイレクトし、failureFlash: trueによってエラーメッセージを自動でフラッシュにセットします。
  • 認証に成功した場合のみ、次のコールバック関数（リダイレクト処理）を実行します。

2. ログイン状態のグローバル化 (index.js)

ログインしているユーザーの情報を、すべてのテンプレートでcurrentUserという変数名で使えるように、ミドルウェアを修正しました。 

app.use((req, res, next) => {
    res.locals.currentUser = req.user; // ログイン中のユーザー情報
    res.locals.success = req.flash('success');
    res.locals.error = req.flash('error');
    next();
});

3. ナビゲーションバーの動的表示 (views/partials/navbar.ejs)

currentUser変数が存在するかどうかで、ナビゲーションバーに表示するリンクを「ログイン/登録」と「ログアウト」で切り替えるようにしました。さらに、ログイン中はユーザー名を表示するように改善しました。 

<% if(!currentUser) { %>
    <a href="/login" class="nav-link">ログイン</a>
    <a href="/register" class="nav-link">ユーザー登録</a>
<% } else { %>
    <span class="navbar-text me-2">こんにちは、<%= currentUser.username %>さん</span>
    <a href="/logout" class="nav-link">ログアウト</a>
<% } %>

これで、ユーザー認証に関する一連の基本フローが完成し、UIもより使いやすくなりました。
