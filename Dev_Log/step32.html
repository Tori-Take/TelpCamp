TelpCamp プロジェクト作成の記録

Step 32: アクセス制御（認可）の実装

認証機能の次のステップとして、特定の操作をログインしているユーザーのみに許可する「アクセス制御（認可）」を実装しました。これにより、ログインしていないユーザーがキャンプ場の登録や編集、削除などを行えないようにし、アプリケーションのセキュリティを向上させました。

学習の目的

  • 認証（Authentication）と認可（Authorization）の違いを理解する。
  • Expressのミドルウェアを作成し、ルートに適用する方法を学ぶ。
  • Passportが提供するreq.isAuthenticated()メソッドを使い、ユーザーのログイン状態をチェックする方法を習得する。
  • 保護したいルートにミドルウェアを組み込み、アクセスを制限する。

実装の解説

ログイン状態をチェックするミドルウェアを作成し、それを保護したいルートに適用するという流れで実装しました。

1. ログインチェックミドルウェアの作成 (middleware.js)

ユーザーがログインしているかどうかを判定するisLoggedInというミドルウェアを新しく作成しました。このミドルウェアは、今後の拡張性も考慮してmiddleware.jsという専用ファイルに記述しました。 

// middleware.js
module.exports.isLoggedIn = (req, res, next) => {
    // Passportが提供するisAuthenticated()でログイン状態をチェック
    if (!req.isAuthenticated()) {
        // ログインしていなければ、エラーメッセージをフラッシュに保存してログインページへリダイレクト
        req.flash('error', 'この操作を行うにはログインしてください。');
        return res.redirect('/login');
    }
    // ログインしていれば、次の処理（ルートハンドラなど）へ進む
    next();
}

💡 認証と認可

認証 (Authentication)は「あなたが誰であるか」を確認するプロセスです（例：ログイン）。 一方、認可 (Authorization)は「あなたに何をする権限があるか」を決定するプロセスです。今回のisLoggedInミドルウェアは、ログインしているユーザーにのみ特定の操作を許可するため、「認可」の役割を担っています。

2. ルートへのミドルウェア適用

作成したisLoggedInミドルウェアを、保護が必要なキャンプ場とレビューのルートに適用しました。 

// routes/campgrounds.js
const { isLoggedIn } = require('../middleware');

// キャンプ場登録フォームの表示
router.get('/new', isLoggedIn, (req, res) => { ... });

// キャンプ場の作成
router.post('/', isLoggedIn, validateCampground, wrapAsync(async (req, res) => { ... }));

// ...編集、更新、削除ルートも同様に適用

これにより、ログインしていないユーザーがこれらのURLに直接アクセスしようとしても、ログインページにリダイレクトされるようになり、アプリケーションの安全性が確保されました。
