TelpCamp プロジェクト作成の記録

Step 33: ユーザー体験(UX)の向上（自動ログインとリダイレクト）

認証機能をより洗練させ、ユーザーにとってストレスのないスムーズな体験を提供するために、2つの重要な改善を行いました。「登録後の自動ログイン」と「ログイン後のインテリジェントなリダイレクト」です。

学習の目的

  • req.login()メソッドを使い、ユーザー登録直後に自動でログイン状態にする方法を確認する。
  • セッションを利用して、ログイン前にアクセスしようとしていたURLを一時的に保存する方法を学ぶ。
  • ログイン後、固定のページではなく、元々ユーザーが目指していたページにリダイレクトさせることで、ユーザー体験を向上させる。
  • PassportのkeepSessionInfo: trueオプションの役割を理解する。

実装の解説

1. 登録後の自動ログイン

ユーザー登録処理の最後にPassportが提供するreq.login()メソッドを呼び出すことで、ユーザーは登録後に再度ログインする手間なく、シームレスにサービスを使い始めることができます。 

// routes/users.js
router.post('/register', catchAsync(async (req, res, next) => {
    try {
        // ... ユーザー登録処理 ...
        const registeredUser = await User.register(user, password);
        // 登録したユーザーで自動的にログインさせる
        req.login(registeredUser, err => {
            if (err) return next(err);
            // ... リダイレクト処理 ...
        });
    } catch (e) {
        // ... エラー処理 ...
    }
}));

2. ログイン後のインテリジェントなリダイレクト

ユーザーがログインを求められた後、元々アクセスしようとしていたページに戻れるように実装を改善しました。

💡 実装の流れ

  1. URLの保存: middleware.jsのisLoggedInミドルウェアで、ログインしていないユーザーを検出した際に、アクセスしようとしていたURL(req.originalUrl)をセッションにreq.session.returnToとして保存します。 
    // middleware.js
if (!req.isAuthenticated()) {
    req.session.returnTo = req.originalUrl; // 元のURLを保存
    req.flash('error', 'この操作を行うにはログインしてください。');
    return res.redirect('/login');
}
  2. リダイレクト先の決定: ログイン処理と登録処理の成功後、セッションにreturnToがあればそのURLへ、なければデフォルトの/campgroundsへリダイレクトします。 
    // routes/users.js
const redirectUrl = req.session.returnTo || '/campgrounds';
delete req.session.returnTo; // 一度使ったら削除
res.redirect(redirectUrl);
  3. セッション情報の維持: ログイン処理でpassport.authenticateを使用する際、keepSessionInfo: trueオプションを追加しました。これにより、万が一ログインに失敗してもreq.session.returnToの情報がセッションから消えるのを防ぎます。 
    router.post('/login', passport.authenticate('local', { ..., keepSessionInfo: true }), ...);

これらの改善により、例えばユーザーがログアウト状態で「キャンプ場登録」ボタンを押し、ログインページに飛ばされた後、ログインを完了すると、自動的にキャンプ場登録ページに戻れるようになりました。これは、ユーザーが「何をしようとしていたか」をアプリケーションが覚えていてくれる、非常に親切な設計です。
