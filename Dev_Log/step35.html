Step 35: レビューとユーザーの紐付け

キャンプ場と同様に、レビューにも「誰が投稿したか」という所有者情報を記録するようにしました。これにより、より詳細なアクセス制御（例：レビューの本人による削除）の基盤が整います。

学習の目的

  • Reviewモデルにもauthorフィールドを追加し、Userモデルと関連付ける。
  • レビュー作成時に、ログイン中のユーザーIDをauthorとして保存する。
  • Mongooseの「ネストされたPopulate」を使い、関連データのさらに関連データを取得する方法を学ぶ。
  • ビューで、ネストされたPopulateで取得した情報を表示する。

実装の解説

キャンプ場の時と同様に、モデルの更新、ルートの修正、そしてビューの更新という流れで実装しました。

1. Reviewモデルへのauthorフィールド追加

models/review.jsを修正し、投稿者のIDを保存するためのauthorフィールドを追加しました。 

// models/review.js
const reviewSchema = new Schema({
    body: String,
    rating: Number,
    author: {
        type: Schema.Types.ObjectId,
        ref: 'User'
    }
});

2. レビュー作成ルートの修正 (routes/reviews.js)

レビューを新規作成するPOSTルートで、新しいreviewオブジェクトにログイン中のユーザーID（req.user._id）をauthorとして設定してから保存するように変更しました。 

// routes/reviews.js
router.post('/', isLoggedIn, ..., wrapAsync(async (req, res) => {
    // ...
    const review = new Review(req.body.review);
    review.author = req.user._id; // ログインユーザーを登録者として設定
    // ...
}));

3. ネストされたPopulateの実行 (routes/campgrounds.js)

キャンプ場詳細ページでレビューの投稿者名を表示するために、populateをネストさせる必要がありました。reviewsをpopulateし、さらにその中の各レビューのauthorをpopulateするように指定します。 

// routes/campgrounds.js
const campground = await Campground.findById(id).populate({
    path: 'reviews',
    populate: {
        path: 'author'
    }
}).populate('author');

💡 ネストされたPopulate

populate()の引数にオブジェクトを渡すことで、より詳細な指定が可能です。pathでpopulateしたいフィールド名を指定し、さらにその中のフィールドをpopulateしたい場合は、populateプロパティをネストして記述します。これにより、関連データのさらに先の関連データまで一度に取得できます。

この実装により、レビューにも所有者情報が記録され、アプリケーションのデータ構造がより豊かになりました。
