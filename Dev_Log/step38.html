Step 38: レビューの認可（所有権）

キャンプ場の認可機能に続き、レビューの削除も投稿した本人（所有者）のみに許可する認可機能を実装しました。これにより、他人のレビューを勝手に削除できないようになり、アプリケーションの整合性とセキュリティが向上します。

学習の目的

  • レビューの所有者を確認するための認可ミドルウェアを作成する。
  • ネストされたルート（/campgrounds/:id/reviews/:reviewId）で複数のIDを扱う方法を理解する。
  • クライアントサイドとサーバーサイドの両方で、レビュー削除の権限をチェックする。

実装の解説

キャンプ場の時と同様に、認可ミドルウェアを作成し、それをルートとビューに適用しました。

1. レビュー認可ミドルウェアの作成 (middleware.js)

レビューの所有者と現在ログインしているユーザーが一致するかを検証するisReviewAuthorミドルウェアを作成しました。 

// middleware.js
const Review = require('./models/review');

module.exports.isReviewAuthor = async (req, res, next) => {
    const { id, reviewId } = req.params;
    const review = await Review.findById(reviewId);
    if (!review.author.equals(req.user._id)) {
        req.flash('error', 'その操作を行う権限がありません。');
        return res.redirect(`/campgrounds/${id}`);
    }
    next();
}

2. サーバーサイドでの認可 (routes/reviews.js)

作成したisReviewAuthorミドルウェアを、レビューを削除するDELETEルートに適用しました。 

// routes/reviews.js
const { isLoggedIn, isReviewAuthor } = require('../middleware');

router.delete('/:reviewId', isLoggedIn, isReviewAuthor, ...);

3. クライアントサイドでの認可 (views/campgrounds/show.ejs)

キャンプ場詳細ページで、ログイン中のユーザーがそのレビューの投稿者である場合にのみ、削除ボタンが表示されるようにビューを修正しました。 

<% for(let review of campground.reviews) { %>
    ...
    <% if(currentUser && review.author.equals(currentUser._id)) { %>
        <form ...>
            <button class="btn btn-sm btn-danger">削除</button>
        </form>
    <% } %>
    ...
<% } %>

これで、レビューに対しても堅牢な所有者ベースのアクセス制御が実装できました。
