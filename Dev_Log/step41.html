Step 41: 画像の更新・削除とデータ整合性の確保

Step 40で実装した画像アップロード機能を発展させ、キャンプ場の編集ページで「画像の追加」と「特定の画像の削除」ができるようにしました。 さらに、キャンプ場自体が削除された際に、関連する画像がCloudinary上からも自動で削除されるようにし、データの整合性を保つ仕組みを構築しました。

学習の目的

  • 既存のドキュメントに対して、新しい画像データを追加する更新処理を実装する。
  • チェックボックスを利用して、複数の画像の中から削除したいものを選択し、サーバーに送信する方法を学ぶ。
  • サーバーサイドで、テキスト情報の更新、画像の追加、画像の削除という複数の操作を一つのリクエストで処理するロジックを構築する。
  • Mongooseのミドルウェア（post('findOneAndDelete')）を活用し、データベースのレコード削除と連動して外部サービス（Cloudinary）上のリソースを削除する方法（カスケード削除）を習得する。
  • Joiを使ったサーバーサイドバリデーションを拡張し、新しいフォームデータ（deleteImages）に対応させる方法を理解する。

実装の解説

画像のCRUD（作成、読み取り、更新、削除）操作を完成させるため、アプリケーションの各所を修正・機能追加しました。

  1. 1. 編集フォームの機能拡張 (edit.ejs)

    キャンプ場編集フォームに、画像の「追加」と「削除」の機能を追加しました。削除したい画像のチェックボックスをオンにすると、その画像のfilenameがdeleteImagesという名前の配列としてサーバーに送信されます。 

    <!-- edit.ejs -->
<!-- 新しい画像を追加するための入力欄 -->
<div class="mb-3">
    <label for="image" class="form-label">画像の追加</label>
    <input class="form-control" type="file" id="image" name="image" multiple>
</div>

<!-- 既存の画像と削除用チェックボックス -->
<div class="mb-3">
    <h6>画像の削除</h6>
    <% campground.images.forEach((img, i) => { %>
        <input type="checkbox" name="deleteImages[]" value="<%=img.filename%>">
        <img src="<%= img.url%>" class="img-thumbnail" alt="">
    <% }) %>
</div>

  2. 2. バリデーションスキーマの更新 (schemas.js)

    編集フォームからdeleteImagesという新しいデータが送られてくるため、Joiのバリデーションスキーマを更新して、このデータを許可する必要がありました。これが"deleteImages" is not allowedというエラーの解決策でした。 

    // schemas.js
module.exports.campgroundSchema = Joi.object({
    campground: Joi.object({
        // ...
    }).required(),
    deleteImages: Joi.array().items(Joi.string()) // この行を追加
});

  3. 3. 更新コントローラーのロジック拡充 (controllers/campgrounds.js)

    updateCampgroundコントローラーを更新し、画像の追加と削除を同時に処理できるようにしました。 

    // controllers/campgrounds.js
module.exports.updateCampground = async (req, res) => {
    // ...
    // 1. 新しい画像を追加
    const imgs = req.files.map(f => ({ url: f.path, filename: f.filename }));
    campground.images.push(...imgs);
    await campground.save();

    // 2. チェックされた画像を削除
    if (req.body.deleteImages) {
        // Cloudinaryからファイルを削除
        for (let filename of req.body.deleteImages) {
            await cloudinary.uploader.destroy(filename);
        }
        // DBのimages配列から該当オブジェクトを削除
        await campground.updateOne({ $pull: { images: { filename: { $in: req.body.deleteImages } } } });
    }
    // ...
};

  4. 4. データ整合性の確保 (models/campground.js)

    キャンプ場を削除した際に、関連する画像がCloudinary上に残ってしまう「孤児ファイル」問題を防ぐため、Mongooseのミドルウェアを実装しました。

    💡 カスケード削除の実装

    post('findOneAndDelete')フックは、ドキュメントが削除された後に実行されます。このフック内で、削除されたドキュメント(doc)が持っていた画像情報(doc.images)をループ処理し、Cloudinary上の実ファイルを一つずつ削除します。これにより、データベースとクラウドストレージの状態が常に同期され、データの整合性が保たれます。 

    // models/campground.js
campgroundSchema.post('findOneAndDelete', async function (doc) {
    if (doc) {
        // ... 関連レビューの削除 ...

        // 関連する画像をCloudinaryから削除
        for (const img of doc.images) {
            await cloudinary.uploader.destroy(img.filename);
        }
    }
});

  5. 5. 一覧ページへの画像表示 (index.ejs)

    最後に、キャンプ場一覧ページに各キャンプ場の最初の1枚の画像をサムネイルとして表示するように修正し、視覚的な魅力を高めました。

    これらの実装により、画像のCRUD操作が完成し、アプリケーションの完成度が飛躍的に向上しました。
