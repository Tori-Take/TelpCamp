Step 43: クラスターマップの実装

キャンプ場一覧ページに、Mapbox GL JSを利用したインタラクティブなクラスターマップを実装しました。これにより、多数のキャンプ場の位置を地図上で視覚的に把握できるようになり、ユーザーが目的のキャンプ場を探しやすくなりました。

学習の目的

  • Mapbox GL JSをWebアプリケーションに統合し、地図を表示する方法を学ぶ。
  • サーバーサイドのデータをクライアントサイドのJavaScriptに安全に渡す方法（JSON.stringify）を習得する。
  • GeoJSONデータ構造を理解し、Mapboxのデータソースとして利用する方法を学ぶ。
  • Mapboxのクラスター機能を使って、多数のデータポイントを効率的にグループ化して表示する方法を実践する。
  • マップ上の要素（クラスター、個別のポイント）に対するインタラクション（クリックイベント、ポップアップ表示）を実装する方法を学ぶ。

実装の解説

実装は、Mapboxのライブラリ読み込み、サーバーからのデータ受け渡し、そしてマップを制御するクライアントサイドJavaScriptの作成という流れで行いました。

1. Mapboxライブラリとマップコンテナの準備 (views/campgrounds/index.ejs)

まず、一覧表示ページにMapboxのCSSとJSライブラリをCDN経由で読み込み、マップを表示するためのコンテナ（<div id="cluster-map">）を設置しました。 

<!-- MapboxのCSS -->
<link href="https://api.mapbox.com/mapbox-gl-js/v2.3.1/mapbox-gl.css" rel="stylesheet">
<!-- MapboxのJS -->
<script src="https://api.mapbox.com/mapbox-gl-js/v2.3.1/mapbox-gl.js"></script>

<h1>すべてのキャンプ場</h1>

<!-- マップを表示するコンテナ -->
<div id="cluster-map"></div>

<!-- ... キャンプ場一覧のループ ... -->

2. サーバーサイドからクライアントサイドへのデータ受け渡し (index.ejs)

EJSで受け取ったキャンプ場データ（campgrounds）を、クライアントサイドのJavaScriptで使えるように、JSON.stringifyを使ってJSON文字列に変換して埋め込みました。

💡 安全なデータ受け渡し

<%- JSON.stringify(campgrounds) %> のように、<%= %>ではなく<%- %>を使用しています。これは、JSON.stringifyが生成するクォーテーションマークなどの特殊文字がHTMLエスケープされるのを防ぐためです。これにより、JavaScriptは正しいJSONとしてデータを解釈できます。 

<script>
    // MapboxのアクセストークンをEJS経由で設定
    const mapToken = '<%- process.env.MAPBOX_TOKEN %>';
    // キャンプ場データをJSONとしてJavaScript変数に格納
    const campgrounds = { features: <%- JSON.stringify(campgrounds) %> };
</script>
<!-- 実際にマップを制御する外部JSファイル -->
<script src="/javascripts/clusterMap.js"></script>

3. クラスターマップのロジック実装 (public/javascripts/clusterMap.js)

マップの初期化、データソースの追加、クラスターレイヤーの定義、インタラクションの追加など、マップに関する主要なロジックをこのファイルに記述しました。 

// public/javascripts/clusterMap.js

mapboxgl.accessToken = mapToken;
const map = new mapboxgl.Map({
    container: 'cluster-map',
    style: 'mapbox://styles/mapbox/light-v10',
    center: [-103.5917, 40.6699],
    zoom: 3
});

map.on('load', () => {
    // 1. GeoJSONデータソースを追加
    map.addSource('campgrounds', {
        type: 'geojson',
        data: campgrounds,
        cluster: true, // クラスター機能を有効化
        clusterMaxZoom: 14,
        clusterRadius: 50
    });

    // 2. クラスター（円）のレイヤーを追加
    map.addLayer({
        id: 'clusters',
        type: 'circle',
        source: 'campgrounds',
        filter: ['has', 'point_count'], // クラスター化されたポイントのみ対象
        paint: { /* ...円の色やサイズを定義... */ }
    });

    // 3. クラスター内の件数を表示するレイヤーを追加
    map.addLayer({
        id: 'cluster-count',
        type: 'symbol',
        source: 'campgrounds',
        filter: ['has', 'point_count'],
        layout: {
            'text-field': '{point_count_abbreviated}',
            'text-font': ['DIN Offc Pro Medium', 'Arial Unicode MS Bold'],
            'text-size': 12
        }
    });

    // 4. クラスター化されていない個別のポイントのレイヤーを追加
    map.addLayer({
        id: 'unclustered-point',
        type: 'circle',
        source: 'campgrounds',
        filter: ['!', ['has', 'point_count']], // クラスター化されていないポイントのみ対象
        paint: { /* ...円の色やサイズを定義... */ }
    });

    // 5. クラスタークリック時のズーム処理
    map.on('click', 'clusters', (e) => { /* ... */ });

    // 6. 個別ポイントクリック時のポップアップ表示処理
    map.on('click', 'unclustered-point', (e) => { /* ... */ });

    // 7. マウスカーソルの変更処理
    map.on('mouseenter', 'clusters', () => { map.getCanvas().style.cursor = 'pointer'; });
    map.on('mouseleave', 'clusters', () => { map.getCanvas().style.cursor = ''; });
});

この実装により、キャンプ場一覧ページは単なるリスト表示から、地理的な情報を活用できるインタラクティブなページへと大きく進化しました。
