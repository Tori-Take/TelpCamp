TelpCamp プロジェクト作成の記録

Step 44: 本番環境への第一歩 - MongoDB Atlasへの接続

いよいよアプリケーションのデプロイ（公開）に向けて、最初の大きなステップに進みます。これまでは開発用PCの中にあるローカルのMongoDBを使用していましたが、これをインターネット経由でどこからでもアクセスできるクラウドデータベース「MongoDB Atlas」に移行します。

学習の目的

  • ローカルDBからクラウドDBへ移行する目的とメリットを理解する。
  • MongoDB Atlasの基本的な用語（プロジェクト、クラスター、データベースユーザー）の役割を整理する。
  • 環境変数（.envファイル）を使い、機密情報である接続文字列を安全に管理する方法を学ぶ。
  • 接続文字列にデータベース名を正しく指定し、意図したデータベースに接続する方法を習得する。

実装の解説

1. MongoDB Atlasのセットアップ

まず、MongoDB Atlasのサイトでデータベースの器を用意します。

💡 Atlasの階層構造

  1. プロジェクト: アプリケーション全体を管理する一番大きな箱。今回は「TelpCamp-Prod」のような名前で作成しました。
  2. クラスター: 実際にデータベースが動くサーバー群。無料枠の「M0」で「telpcampcluster」を作成しました。
  3. データベースユーザー: アプリケーションがデータベースに接続するためのIDとパスワード。Atlasの管理画面にログインする人間（プロジェクトメンバー）とは全く別物です。
  4. ネットワークアクセス: どこからの接続を許可するかの設定。開発中は「ALLOW ACCESS FROM ANYWHERE」(0.0.0.0/0)に設定しました。

2. アプリケーション側の設定

Atlasで取得した接続文字列を、アプリケーションに安全な方法で設定します。

  1. .envファイルの作成:

    パスワードなどの機密情報をコードに直接書くのは非常に危険です。そこで、プロジェクトのルートに.envファイルを作成し、そこに接続文字列を記述します。

    # .env
DB_URL=mongodb+srv://<ユーザー名>:<パスワード>@telpcampcluster.xxxxx.mongodb.net/telp-camp?retryWrites=true...

    💡 データベース名の指定が最重要！

    接続文字列の.net/と?retryWritesの間に、使用したいデータベース名（今回は/telp-camp）を必ず指定します。これを忘れると、MongoDBはデフォルトのtestという名前のデータベースを自動的に使用してしまいます。今回の作業で、実際にデータがtestデータベースに保存されてしまう現象に遭遇し、この指定の重要性を実感しました。

  2. .gitignoreの更新:

    機密情報が含まれる.envファイルがGitリポジトリにアップロードされないように、.gitignoreに.envを追記します。 

    # .gitignore
node_modules/
.env
  3. index.jsの修正:

    dotenvパッケージが.envファイルを読み込み、その値をprocess.env.DB_URLとして利用できるようにします。そして、データベース接続部分を以下のように修正しました。 

    // index.js
// .envファイルから環境変数を読み込む (開発環境でのみ使用)
if (process.env.NODE_ENV !== 'production') require('dotenv').config();

// ...

async function startServer() {
    // 環境変数にDB_URLがあればそれ(Atlas)を使い、なければローカルDBを使う
    const dbUrl = process.env.DB_URL || 'mongodb://127.0.0.1:27017/telp-camp';
    try {
        await mongoose.connect(dbUrl);
        console.log("MongoDBに接続しました。");
        // ...
    }
}

    この設定により、.envファイルがあればAtlasに、なければ（またはコメントアウトすれば）従来のローカルDBに接続先を簡単に切り替えられるようになりました。

3. 動作確認

サーバーを起動(node index.js)し、コンソールに「MongoDBに接続しました。」と表示されることを確認。その後、実際にキャンプ場やユーザーを登録し、Atlasの管理画面の「Browse Collections」から、意図したプロジェクトのtelp-campデータベースにデータが保存されていることを確認して、移行作業は完了です！
