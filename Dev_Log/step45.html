Step 45: セッションの永続化 (connect-mongo)

デプロイに向けた重要な改善として、セッション情報の保存先をアプリケーションのメモリからMongoDB Atlasに変更しました。

学習の目的

  • デフォルトのメモリストアの問題点（サーバー再起動でセッションが消える）を理解する。
  • connect-mongoを使い、セッション情報をデータベースに永続化する方法を学ぶ。
  • サーバーを再起動してもユーザーのログイン状態が維持されることを確認する。

実装の解説

1. connect-mongoのインストール

セッション情報をMongoDBに保存するためのパッケージをインストールします。 

npm install connect-mongo

2. index.jsの修正

express-sessionの設定を変更し、保存先（ストア）としてconnect-mongoを指定します。 

// index.js
const MongoStore = require('connect-mongo');

// ...

const dbUrl = process.env.DB_URL || 'mongodb://127.0.0.1:27017/telp-camp';

const store = MongoStore.create({
    mongoUrl: dbUrl,
    touchAfter: 24 * 60 * 60, // 24時間
    crypto: {
        secret: process.env.SESSION_SECRET || 'mysecret'
    }
});

const sessionConfig = {
    store, // 保存先として作成したMongoStoreインスタンスを指定
    name: 'session', // セキュリティ向上のためクッキー名を変更
    secret: process.env.SESSION_SECRET || 'mysecret',
    // ...
};

app.use(session(sessionConfig));

💡 なぜサーバーを止めても大丈夫になったのか？

これまでのセッション情報は、サーバープロセスのメモリ上に保存されていました。そのため、サーバーを停止するとメモリ上のデータは全て消えてしまいます。
今回の変更で、セッション情報は外部のMongoDB Atlas上に保存されるようになりました。データベース上のデータはサーバーを停止しても消えません。サーバーが再起動すると、再びAtlasに接続し、そこに保存されているセッション情報を読み込むため、ログイン状態が維持されるのです。

3. 動作確認

実際にアプリケーションにログインした後、ターミナルでサーバーを停止（Ctrl+C）し、再度起動（node index.js）。ブラウザをリロードしてもログイン状態が維持されていることを確認しました。また、MongoDB Atlasの管理画面で、telp-campデータベース内にsessionsという新しいコレクションが作成されていることも確認できました。
